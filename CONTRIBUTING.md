# Contributing to HumanGuard

## Getting Started

```bash
git clone https://github.com/<your-fork>/HumanGuard
cd HumanGuard
pip install -r requirements.txt
python -m backend.app        # start local server on :5050
pytest tests/ -v             # run the full test suite (158 tests)
```

No linter or formatter is currently configured. Follow the style of the surrounding code.

## Making Changes

1. Fork the repository and create a branch from `main`.
2. Make your changes. Keep commits focused — one logical change per commit.
3. Add or update tests for any behavior you change (see `tests/`).
4. Run `pytest tests/ -v` and confirm all tests pass before opening a PR.
5. Open a pull request against `main` with a clear description of what changed and why.

## Test Coverage

The project maintains 158 tests across 15 files. New features should come with tests. Bug fixes should include a regression test where practical.

## Adding Features

- New behavioral features must be added to `backend/features/feature_extractor.py` and the `feature_names.json` artifact regenerated by running `python -m models.run_training`.
- API endpoints should follow the existing Flask route pattern and include input validation.
- Database schema changes belong in `backend/db/schema.sql`; run `python -m backend.db.migrate` locally to apply them.

## Security

Please review `SECURITY.md` before submitting changes that touch authentication, API keys, webhooks, or input validation. Do not introduce hardcoded credentials, raw SQL string interpolation, or unauthenticated access to scored endpoints.
